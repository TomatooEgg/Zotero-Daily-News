# 输出契约（必须严格遵守）
- 只输出一个 JSON 对象：{"sections":[...], "key_terms":[...]}
- 不要用 markdown 代码块包裹整个 JSON；不要输出 JSON 以外的任何文字
- 每个 section 含 "heading" 与 "body"；body 是字符串，换行写为 \n（两个字符）
- sections 必须恰好 5 项，heading 固定为以下 5 个（顺序一致）：
  1. 研究背景与问题
  2. 核心方法与透明度
  3. 主要发现与机制推导
  4. 跨层审计与错位排查
  5. 局限与启示
- key_terms：5-8 项，格式「中文[English]」

# 格式硬规则（违反会导致页面渲染失败）
R1. Mermaid（放在「主要发现与机制推导」body 末尾）：
  - 围栏：\n\n```mermaid\ngraph TD\n...\n```\n
  - 节点写法：A["短标签"] --> B["短标签"]（直引号，禁止写 \" 或 [\"）
  - 节点文本 ≤30 字；节点内不写公式、不换行
R2. 表格（放在「核心方法与透明度」body 内）：
  - GFM 四列表，表头：| 理论构念 | 代理变量 | 数据来源 | 处理逻辑 |
  - 表格前必须有空行（\\n\\n）
R3. 数学符号：
  - 带下标/上标必须用 $...$，如 $\\bar{\\gamma}_W$、$\\mathrm{Cov}(\\gamma)^{-1}$
  - 禁止裸写 γ̄_W、γ̄_{...} 等含 _ 的伪 LaTeX
  - 简单符号可用 Unicode：α、≥、→
R4. 排版：每段用 - 或 1. 拆解；严禁超 3 句长段；**加粗**核心变量、关键数据与结论

- 宏观：制度、文化、结构性分布结果
